## 3. 数据流

### 上传分析流程
```
用户上传文件 → /api/parse (解析文本)
  → 存入 IndexedDB (textbooks 表)
  → /api/analyze/textbook (提取章节)
  → 存入 IndexedDB (chapters 表)
  → /api/analyze/chapter (逐章分析知识点 + 总结)
  → 存入 IndexedDB (knowledgePoints 表, 更新 chapters 表)
  → 用户查看知识树 / 思维导图
```

### 对话流程
```
用户输入问题
  → 从 IndexedDB 搜索相关知识点 (关键词匹配)
  → POST /api/chat (带上下文)
  → Claude 流式返回
  → 实时渲染 + 存入 IndexedDB
```

## 4. 数据库设计

6 张表，详见 `API接口文档.md`：
- `textbooks` — 教材
- `chapters` — 章节
- `knowledgePoints` — 知识点（树状结构，parentId 自引用）
- `mindMaps` — 思维导图缓存
- `chatSessions` — 对话会话
- `chatMessages` — 对话消息

## 5. API 设计

5 个端点，详见 `API接口文档.md`：
- `POST /api/parse` — 文件解析
- `POST /api/analyze/textbook` — 提取章节结构
- `POST /api/analyze/chapter` — 分析单个章节
- `POST /api/mindmap` — 生成思维导图
- `POST /api/chat` — 流式对话

## 6. 关键决策

1. **无状态服务端**：API 只做计算，不存数据，数据全在客户端 IndexedDB
2. **混合文件解析**：PDF/Word 服务端解析，图片 OCR 客户端执行（保护隐私）
3. **增量分析**：先分章，再逐章分析知识点，显示进度条
4. **流式聊天**：SSE 实现打字效果
5. **关键词匹配上下文**：避免引入向量数据库，简单高效
